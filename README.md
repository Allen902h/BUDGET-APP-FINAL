# BUDGET-APP-FINAL

A Laravel 10 budget management system with:

- dashboard-based budget tracking
- income cycles
- categories and transactions
- savings goals
- profile management
- Excel report export
- responsive Blade UI with custom animations

## Stack

- PHP 8.1+
- Laravel 10
- MySQL / MariaDB
- Blade, CSS, JavaScript
- Vite tooling available for frontend assets

## Local Setup

1. Clone the repository.
2. Install PHP dependencies:

```bash
composer install
```

3. Copy the environment file:

```bash
cp .env.example .env
```

4. Set your database credentials in `.env`.
5. Generate the app key:

```bash
php artisan key:generate
```

6. Run migrations:

```bash
php artisan migrate
```

7. Start the app:

```bash
php artisan serve
```

Optional frontend commands:

```bash
npm install
npm run dev
```

## Production / Laravel Cloud Notes

This repository is a standard Laravel application and should be imported from the repository root.

Important:

- Make sure Laravel Cloud imports the branch that contains the Laravel app files at the repository root.
- If Laravel Cloud shows `Unsupported framework`, re-import the repository after confirming the correct default branch.

## Recommended Laravel Cloud Environment Variables

Set these in Laravel Cloud:

- `APP_NAME=BUDGET-APP-FINAL`
- `APP_ENV=production`
- `APP_DEBUG=false`
- `APP_URL=https://your-domain`
- `APP_KEY=` generated by Laravel Cloud or your deploy process
- `DB_CONNECTION=mysql`
- `DB_HOST=...`
- `DB_PORT=3306`
- `DB_DATABASE=...`
- `DB_USERNAME=...`
- `DB_PASSWORD=...`
- `CACHE_DRIVER=file` or Redis if provisioned
- `SESSION_DRIVER=file` or database / Redis if configured
- `QUEUE_CONNECTION=sync` unless you add a worker

## Deploy Checklist

Before creating the Laravel Cloud app:

1. Confirm the repository root contains `artisan`, `composer.json`, `app/`, `routes/`, and `public/`.
2. Confirm the default GitHub branch points to the uploaded Laravel code.
3. Reconnect or re-import the repository in Laravel Cloud if it previously scanned the wrong branch.
4. Configure environment variables.
5. Attach a MySQL database resource.
6. Run migrations during or after the first deploy.
